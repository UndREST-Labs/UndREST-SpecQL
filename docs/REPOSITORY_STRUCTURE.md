# Repository Structure Guide

This document describes the organization of the SpecRecon repository.

SpecRecon is a suite of API spec and API request reconnaissance tools. The two current components are **SpeQL** (API Spec Query Analyser) and **APISpy** (DevTools browser extension).

## Root Directory

The root directory contains only essential files for quick access:

- **README.md** - Main project documentation
- **CONTRIBUTING.md** - Contributing guidelines
- **LICENSE** - License information
- **SpeQL.py** - CLI entry point for the SpeQL query engine
- **setup.sh** - Automated setup script
- **refresh-database.sh** - Bash script to refresh database from Azure specs
- **refresh_database.py** - Cross-platform Python script for database refresh
- **analyze.py** - Python-based security analyzer (no dependencies)
- **run-queries.sh** - CodeQL query execution script

## Key Changes from Previous Structure

### What Changed

1. **Removed implementation documentation** - Removed IMPLEMENTATION_NOTES.md, IMPLEMENTATION_SUMMARY.md, IMPLEMENTATION_SUMMARY_FIX.md, and QUERY_FIXES_SUMMARY.md as they were temporary development notes.

2. **Created docs/ directory** - Moved all documentation except README.md and CONTRIBUTING.md to docs/ to keep the root clean.

3. **Fixed database directory handling** - Both refresh scripts now create the database parent directory if it doesn't exist, fixing the issue where Git can't track empty directories.

4. **Updated .gitignore** - The entire database/ directory is now ignored since it's generated by refresh scripts.

### What Stayed the Same

- CodeQL query for SAS URI detection (SasUriInResponse.ql) is retained
- Python analyzer (analyze.py) is retained with all security checks
- Core functionality scripts remain in the root for easy access
- Scripts directory structure is unchanged
- Query pack configuration is unchanged

## Benefits of New Structure

1. **Cleaner root directory** - Essential files only, easier to navigate
2. **Better organization** - Documentation grouped logically in docs/
3. **Improved robustness** - Scripts handle missing directories gracefully
4. **Maintained functionality** - All queries and features work as before
